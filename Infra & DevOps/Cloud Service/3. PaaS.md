# 개념
IaaS는 컴퓨터를 빌린 다음에 내가 쓸 것들을 하나하나 정해줘야 하는 반면, PaaS(Platform as a Service)는 이름에서 부터 알수 있듯이 **이미 개발에 필요한 모든 인프라 세팅이 끝난 플랫폼을 빌려 쓰는 느낌**이다.
# 특징
- 가상 컴퓨터 빌리기, 운영체제 설치, 자바/파이썬/Node.js 환경 세팅, 네트워크 및 방화벽 설정 등이 공급업체에 의해 이미 다 되어 있다.
- 따라서 개발자는 인프라 환경에는 신경을 쓸필요 없이, 내가 짠 코드를 올리기만 하면 된다.
# 장점
1. **딸깍 배포:** 
	- 리눅스 명령어 입력하고, Nginx 세팅하고, 도커 이미지 빌드할 필요가 전혀 없다. 내가 만든 프론트엔드 코드를 Github에 올리기만 하면, PaaS(Vercel 등)가 알아서 빌드하고 배포해준다.
2. **유지보수 해방 (독박 육아 끝):
	- ** OS 보안 업데이트, HTTPS 보안 인증서(SSL) 발급 및 주기적인 갱신 등을 클라우드 회사가 알아서 다 해준다. 개발자는 새벽에 서버 터질까 봐 잠 못 이룰 일이 없다.
    
3. **가성비와 모니터링 편의성:** 
	- 방문자가 얼마나 오는지, 서버에 에러가 없는지 예쁜 대시보드로 다 보여주고, 초기 단계나 개인 프로젝트 수준에서는 **무료 플랜**을 혜려하게 제공하는 곳이 많아서 돈이 안 든다.
