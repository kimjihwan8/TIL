# 2. workspace

---

## 2.1 개념

workspace는 하나의 저장소 안에 있는 여러개의 패키지를 패키지 관리자가 하나의 프로젝트처럼 인식하고 관리하게 해주는 기능이다.

즉, 여러 프로젝트를 묶고 의존성을 한번에 설치하고 로컬 패키지를 서로 연결해 사용할 수 있게 해주는 기능이다.

모노레포에서 workspace는 각각의 프로젝트들이 서로 남남인 폴더가 아니라 같은 시스템에 속한 패키지들이라는 것을 알려주는 핵심 역할을 한다.

## 2.2 장점

1. 의존성 관리 효율이 높아진다. 각 프로젝트마다 따로 설치할 필요 없이 루트에서 단 한번만 설치하면 된다.
2. 로컬 패키지를 바로 사용할 수 있다. 공통 ui나 유틸 패키지를 npm에 배포하지 않고 같은 저장소 안에서 즉시 import해서 사용할 수 있다.

## 2.3 단점

1. 의존성 구조가 눈에 보이지 않게 복잡해진다. 루트에서 한 번에 설치되기 때문에 각 프로젝트가 실제로 어떤 의존성을 쓰는지 헷갈릴 수 있다.
2. 설정 실수가 전체에 영향을 준다. workspace 설정을 잘못하면 여러 프로젝트의 의존성 해석이 동시에 깨질 수 있다.
3. 코드를 조금만 고쳐도 모든 프로젝트를 매번 처음부터 끝까지 다 빌드해야 했다.
