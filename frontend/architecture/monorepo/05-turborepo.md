# 5. turborepo

---

## 5.1 개념

Rust 언어로 작성된 초고속 모노레포 빌드 도구이다. 설정이 매우 단순하며, 프로젝트 규모가 커져도 초기와 같은 속도를 유지해준다.

## 5.2 나오게 된 이유

1. Lerna의 느린 속도: 1세대 도구들은 순차적으로 하나씩 빌드했기 때문에 패키지가 많아지면 배포가 너무 느렸다.
2. Nx의 과한 복잡도: Nx는 강력하지만 설정이 너무 방대하고 무거웠다. 설정은 기존대로(pnpm/npm) 유지하면서 빌드만 빨라졌으면 좋겠다는 취지로 만들어 졌다.
3. 중복 작업의 피로: 팀원이 이미 빌드한 결과물을 내 컴퓨터에서 또 기다려야 하는 비효율을 해결하고 싶었다.

## 5.3 장점

1. 압도적인 캐싱: 로컬뿐만 아니라 클라우드에 빌드 결과물을 저장한다. 동료가 빌드한 적이 있다면, 다운로드만 받으면 끝난다.
2. 병렬 실행: CPU 자원을 최대한 활용하여 여러 패키지를 동시에 빌드한다. 의존성 순서를 스스로 계산하여 가장 효율적인 순서로 작업을 처리한다.
3. 설정의 단순함: `turbo.json` 파일 하나만 만들면 바로 적용된다. 기존 워크스페이스 구조를 거의 건드리지 않는다.
4. Rust 기반: 도구 자체의 구동 속도가 매우 빠르다.

## 5.4 단점

1. 기능의 단순함: Nx처럼 코드 생성기나 정교한 플러그인 시스템이 없습니다. 오직 빌드 최적화에만 집중되어 있다.
2. 유연성 부족: 정해진 파이프라인 내에서 움직여야 하므로, 매우 특수한 배포 환경에서는 설정이 까다로울 수 있다.
